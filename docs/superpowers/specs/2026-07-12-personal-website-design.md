# westbrooguan 个人技术网站设计

## 目标

为管巍创建一个面向行业技术同行的中文个人网站，并发布到 `westbrooguan.github.io`。网站应突出安全算法、搜索反黑反诈治理、内容风控与行业协作能力，呈现方式参考 Apple 官网的留白、层级、网格和克制动效，但不复制具体页面或品牌资产。

## 受众与信息原则

- 主要受众：安全、风控、搜索治理与算法领域的技术同行和潜在合作伙伴。
- 语言：中文；必要的技术缩写保留英文原文。
- 公开信息：仅展示 `westbrooguan@qq.com`。
- 隐私：不展示手机号、出生年月或其他未确认的个人信息。
- 内容来源：以用户提供的《管巍工作简历》为事实依据，不补写未经确认的任职信息、论文名称或专利名称。

## 视觉方向

采用用户选定的 B 方案“技术作品档案”。

- 色彩：温润纸张灰作为页面底色，墨黑作为正文与高对比区块，暖白作为内容表面，沉静安全绿作为少量状态与重点色。
- 字体：使用 Apple 系统字体栈，中文优先 `PingFang SC`，不加载外部字体。
- 布局：桌面端采用精密网格和较宽留白，首屏形成主张与履历的非对称分栏；移动端按阅读顺序自然堆叠。
- 组件：导航、主张区、四项指标、三个案例、经历时间线、教育与能力、联系入口。
- 动效：仅使用轻量入场、导航状态和卡片悬停；遵守 `prefers-reduced-motion`。
- 质感：编辑化、理性、克制，避免高饱和渐变、玻璃拟态、复杂 3D 和装饰性图片。

## 技术方案

- 站点形态：纯静态单页，无后端、数据库、登录或运行时外部数据。
- 实现原则：复用项目初始化后的最小构建结构；优先原生 HTML、CSS 与少量 JavaScript/React，不增加与单页展示无关的依赖。
- 响应式：桌面、平板和手机三档自然重排；不出现横向滚动或文字裁切。
- 可访问性：语义化区块、清晰的焦点样式、可读对比度、键盘可用锚点、装饰元素不干扰读屏。
- SEO：中文标题、描述、规范链接、Open Graph 基础信息和站点图标。
- 部署：构建静态产物，通过 GitHub Pages 发布到仓库对应的 `github.io` 域名；配置自动化发布流程。

## 验收标准

- 页面完整呈现上述已确认内容，数字和时间与简历一致。
- 页面不显示手机号和出生年月。
- 设计与 B 方案一致：温润浅灰、精密网格、案例优先、克制动效。
- 桌面和移动端均无溢出、遮挡或不可读文本。
- 锚点导航和邮箱链接有效。
- 在减少动态效果偏好下关闭非必要动画。
- 生产构建成功，部署流程可重复执行，公开 URL 可访问。
